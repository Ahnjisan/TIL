# 프론트엔드 기술 스택과 SPA 구조

## 1. 프론트엔드란?

프론트엔드는 브라우저를 통해 사용자에게 보여지는 화면을 기획하고 구현하는 영역이다.

사용자는 서버나 데이터베이스를 직접 보는 것이 아니라, 브라우저에 표시되는 화면을 통해 서비스를 이용한다.

프론트엔드는 다음과 같은 역할을 담당한다.

| 역할 | 설명 |
|---|---|
| 화면 구성 | 사용자가 보는 웹 페이지 구조를 만든다 |
| 스타일 적용 | 화면의 색상, 배치, 크기 등을 꾸민다 |
| 이벤트 처리 | 버튼 클릭, 입력, 화면 전환 등을 처리한다 |
| API 호출 | 백엔드 서버에 데이터를 요청한다 |
| 데이터 표시 | 서버 응답 결과를 화면에 보여준다 |

즉, 프론트엔드는 사용자가 웹 서비스를 직접 사용할 수 있도록 화면과 동작을 구현하는 영역이다.

## 2. HTML

HTML은 HyperText Markup Language의 약자이다.

HTML은 웹 페이지의 구조를 담당한다.  
쉽게 말해 웹 페이지의 뼈대를 만드는 기술이다.

예를 들어 다음과 같은 요소들이 HTML로 구성될 수 있다.

- 제목
- 문단
- 버튼
- 입력창
- 이미지
- 표
- 목록
- 영역 구분

HTML은 태그를 사용하여 구조를 표현한다.

예시는 다음과 같다.

```html
<h1>웹 서비스</h1>
<p>HTML은 웹 페이지의 구조를 담당한다.</p>
<button>클릭</button>
```

HTML에는 의미를 가진 시맨틱 태그도 존재한다.

대표적인 시맨틱 태그는 다음과 같다.

| 태그 | 의미 |
|---|---|
| `<header>` | 페이지나 영역의 머리말 |
| `<nav>` | 메뉴 또는 내비게이션 |
| `<main>` | 주요 콘텐츠 |
| `<section>` | 하나의 구역 |
| `<article>` | 독립적인 콘텐츠 |
| `<footer>` | 페이지나 영역의 하단 |

시맨틱 태그를 사용하면 웹 페이지의 구조와 의미를 더 명확하게 표현할 수 있다.

## 3. CSS

CSS는 Cascading Style Sheets의 약자이다.

CSS는 HTML로 만든 구조에 디자인을 적용하는 기술이다.  
즉, 웹 페이지의 스타일링을 담당한다.

CSS를 사용하면 다음과 같은 요소를 조정할 수 있다.

- 글자 색상
- 배경 색상
- 글자 크기
- 여백
- 정렬
- 버튼 모양
- 화면 배치

예시는 다음과 같다.

```css
h1 {
  color: blue;
  font-size: 24px;
}

button {
  background-color: black;
  color: white;
}
```

HTML이 웹 페이지의 뼈대라면, CSS는 그 뼈대를 보기 좋게 꾸며주는 역할을 한다.

## 4. JavaScript

JavaScript는 웹 페이지에 동적인 기능을 추가하는 기술이다.

HTML과 CSS만으로는 정적인 화면을 만들 수 있지만, 사용자의 동작에 반응하는 기능을 만들기 어렵다.  
JavaScript를 사용하면 버튼 클릭, 입력값 처리, 화면 변경, API 호출과 같은 동작을 구현할 수 있다.

예시는 다음과 같다.

```javascript
const button = document.querySelector("button");

button.addEventListener("click", function () {
  alert("버튼이 클릭되었습니다.");
});
```

정리하면 프론트엔드의 기본 기술은 다음과 같다.

```text
HTML → 구조
CSS → 스타일
JavaScript → 동작
```

## 5. HTML, CSS, JavaScript의 관계

웹 페이지는 보통 HTML, CSS, JavaScript가 함께 동작하면서 만들어진다.

| 기술 | 역할 |
|---|---|
| HTML | 웹 페이지의 구조 |
| CSS | 웹 페이지의 스타일 |
| JavaScript | 웹 페이지의 동작 |

전체 흐름은 다음과 같다.

```text
HTML로 구조 작성
→ CSS로 디자인 적용
→ JavaScript로 동작 구현
→ 브라우저에서 화면 표시
```

예를 들어 로그인 화면을 만든다면 다음과 같이 역할을 나눌 수 있다.

```text
HTML → 아이디 입력창, 비밀번호 입력창, 로그인 버튼 생성
CSS → 입력창과 버튼 디자인
JavaScript → 로그인 버튼 클릭 시 서버에 요청
```

## 6. JSP와 ASP

HTML은 기본적으로 정적인 웹 페이지를 만든다.

하지만 HTML에 서버 측 언어의 코드가 추가되면 동적인 웹 페이지를 만들 수 있다.

대표적인 예시가 JSP와 ASP이다.

| 기술 | 설명 |
|---|---|
| JSP | Java 기반 동적 웹 페이지 기술 |
| ASP | Microsoft 계열 기반 동적 웹 페이지 기술 |

JSP는 Java Server Pages의 약자이다.  
HTML 안에서 Java 코드를 함께 사용할 수 있는 기술이다.

즉, JSP는 HTML을 기반으로 하지만 Java 문법을 함께 사용하여 동적인 화면을 만들 수 있다.

정리하면 다음과 같다.

```text
HTML → 정적인 웹 페이지
JSP → Java 코드가 포함된 동적인 웹 페이지
ASP → Microsoft 계열 기술이 포함된 동적인 웹 페이지
```

## 7. 반응형 웹

반응형 웹은 화면 크기에 따라 UI가 자동으로 조정되는 웹 페이지를 의미한다.

사용자는 데스크톱, 태블릿, 스마트폰 등 다양한 기기에서 웹 서비스를 이용한다.  
각 기기는 화면 크기가 다르기 때문에 같은 화면을 그대로 보여주면 사용성이 떨어질 수 있다.

반응형 웹은 이런 문제를 해결하기 위해 사용된다.

```text
데스크톱 화면
→ 넓은 화면에 맞는 레이아웃 표시

스마트폰 화면
→ 좁은 화면에 맞는 레이아웃 표시
```

반응형 웹을 만들 때는 CSS의 미디어 쿼리를 사용할 수 있다.

예시는 다음과 같다.

```css
@media (max-width: 768px) {
  .container {
    width: 100%;
    font-size: 14px;
  }
}
```

미디어 쿼리는 화면 크기 조건에 따라 다른 CSS 스타일을 적용할 수 있게 해준다.

## 8. Bootstrap

Bootstrap은 반응형 웹 페이지를 쉽게 만들 수 있도록 도와주는 CSS 프레임워크이다.

Bootstrap을 사용하면 미리 만들어진 디자인 요소를 활용할 수 있다.

대표적인 요소는 다음과 같다.

- 버튼
- 입력 폼
- 테이블
- 카드
- 그리드 레이아웃
- 내비게이션 바
- 모달

Bootstrap을 사용하면 직접 CSS를 모두 작성하지 않아도 기본적인 반응형 UI를 빠르게 만들 수 있다.

즉, Bootstrap은 프론트엔드 화면 개발에서 스타일링과 반응형 레이아웃 구현을 도와주는 도구이다.

## 9. SPA

SPA는 Single Page Application의 약자이다.

SPA는 하나의 페이지를 기반으로 동작하는 웹 애플리케이션 방식이다.

기존 웹 페이지는 페이지를 이동할 때마다 서버에서 새로운 HTML 페이지를 받아오는 방식이었다.  
하지만 SPA는 처음에 하나의 페이지를 로드한 뒤, 필요한 데이터만 서버에 요청하고 화면 일부를 변경한다.

흐름은 다음과 같다.

```text
초기 페이지 로드
→ 필요한 데이터만 API 요청
→ 응답 데이터 수신
→ 화면 일부 갱신
```

SPA에서는 페이지 전체가 매번 새로고침되지 않는다.  
대신 화면을 구성하는 컴포넌트가 변경되면서 페이지가 전환되는 것처럼 보인다.

## 10. SPA의 장점과 단점

SPA의 장점은 다음과 같다.

- 페이지 전환이 빠르다.
- 필요한 데이터만 요청할 수 있다.
- 사용자 경험이 자연스럽다.
- 서버 부하를 줄일 수 있다.

하지만 단점도 존재한다.

- 초기 로딩 시간이 길어질 수 있다.
- 검색 엔진 최적화가 어려울 수 있다.
- 보안이나 렌더링 방식에 대한 추가 고려가 필요하다.

SPA의 단점을 보완하기 위해 SSR이라는 개념도 함께 사용될 수 있다.

SSR은 Server Side Rendering의 약자이다.  
서버에서 화면을 렌더링한 뒤 클라이언트에게 전달하는 방식이다.

## 11. React

React는 사용자 인터페이스를 만들기 위한 JavaScript 라이브러리이다.

React는 SPA 기반 프론트엔드 개발에서 많이 사용된다.  
React에서는 화면을 컴포넌트 단위로 나누어 개발한다.

컴포넌트는 화면을 구성하는 독립적인 UI 단위이다.

예시는 다음과 같다.

- Header
- Button
- LoginForm
- PostList
- Modal

React에서는 JSX 문법을 사용할 수 있다.  
JSX는 JavaScript 안에서 HTML과 비슷한 구조를 작성할 수 있게 해주는 문법이다.

예시는 다음과 같다.

```jsx
function Button() {
  return <button>클릭</button>;
}
```

React는 UI 구현에 필요한 기능을 중심으로 제공하기 때문에 라이브러리 성격이 강하다.

## 12. Vue

Vue는 사용자 인터페이스를 만들기 위한 JavaScript 프레임워크이다.

Vue도 React처럼 컴포넌트 기반으로 화면을 구성한다.  
일반적으로 Vue에서는 하나의 컴포넌트 파일 안에 template, script, style을 함께 작성할 수 있다.

예시는 다음과 같다.

```vue
<template>
  <button>클릭</button>
</template>

<script>
export default {
  name: "Button"
};
</script>

<style>
button {
  color: white;
  background-color: black;
}
</style>
```

Vue는 화면 구성뿐만 아니라 애플리케이션 구조와 흐름을 비교적 체계적으로 제공하기 때문에 프레임워크 성격이 강하다.

## 13. 컴포넌트

React와 Vue는 모두 컴포넌트 기반 개발 방식을 사용한다.

컴포넌트는 화면을 구성하는 재사용 가능한 단위이다.

예를 들어 웹 페이지를 다음과 같이 나눌 수 있다.

```text
Header
→ Navigation
→ MainContent
→ PostList
→ Footer
```

컴포넌트 기반으로 개발하면 다음과 같은 장점이 있다.

- 화면을 작은 단위로 나누어 관리할 수 있다.
- 재사용이 쉽다.
- 유지보수가 편해진다.
- 역할 분리가 명확해진다.

즉, 컴포넌트는 복잡한 화면을 작은 UI 단위로 나누어 개발하기 위한 개념이다.

## 14. Library와 Framework

라이브러리와 프레임워크는 모두 개발을 도와주는 도구이다.

하지만 가장 큰 차이는 제어의 흐름이다.

| 구분 | 설명 |
|---|---|
| 라이브러리 | 개발자가 필요한 기능을 직접 호출해서 사용 |
| 프레임워크 | 정해진 구조 안에서 프레임워크가 개발자의 코드를 호출 |

라이브러리는 개발자가 필요한 기능을 가져다 사용하는 방식이다.  
반면 프레임워크는 전체적인 구조와 흐름을 제공하고, 개발자는 그 규칙에 맞춰 코드를 작성한다.

정리하면 다음과 같다.

```text
라이브러리: 개발자 → 라이브러리 호출
프레임워크: 프레임워크 → 개발자 코드 호출
```

React는 라이브러리 성격이 강하고, Vue는 프레임워크 성격이 강하다고 볼 수 있다.

## 15. 전체 흐름 정리

프론트엔드 기술의 흐름은 다음과 같이 정리할 수 있다.

```text
HTML
→ 웹 페이지 구조 생성

CSS
→ 구조에 스타일 적용

JavaScript
→ 사용자 이벤트와 동작 구현

React / Vue
→ 컴포넌트 기반 SPA 개발
```

웹 페이지는 HTML, CSS, JavaScript를 기반으로 만들어진다.  
최근에는 React나 Vue 같은 기술을 활용해 SPA 방식으로 프론트엔드를 개발하는 경우가 많다.

SPA에서는 하나의 페이지를 기반으로 필요한 데이터만 백엔드에 요청하고, 응답 결과를 화면에 반영한다.

## 16. 정리

이번 학습에서는 프론트엔드의 기본 기술 스택과 SPA 구조를 학습했다.

핵심 개념은 다음과 같다.

- 프론트엔드는 브라우저를 통해 사용자에게 보여지는 화면을 구현하는 영역이다.
- HTML은 웹 페이지의 구조를 담당한다.
- CSS는 웹 페이지의 스타일을 담당한다.
- JavaScript는 웹 페이지의 동작을 담당한다.
- JSP는 Java 기반 동적 웹 페이지 기술이다.
- 반응형 웹은 화면 크기에 따라 UI가 조정되는 웹 페이지이다.
- Bootstrap은 반응형 UI 구현을 도와주는 CSS 프레임워크이다.
- SPA는 하나의 페이지를 기반으로 필요한 데이터만 갱신하는 방식이다.
- React와 Vue는 SPA 기반 프론트엔드 개발에서 많이 사용된다.
- React와 Vue는 컴포넌트 기반 개발 방식을 사용한다.
- 라이브러리와 프레임워크는 제어의 흐름에서 차이가 있다.
