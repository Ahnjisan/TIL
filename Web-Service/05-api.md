# 웹 개발 환경 구축

## 1. 개발 환경이 필요한 이유

웹 개발을 하기 위해서는 코드를 작성하고 실행할 수 있는 개발 환경이 필요하다.

개발 환경은 단순히 코드를 입력하는 공간이 아니라, 파일을 관리하고, 코드를 실행하고, 오류를 확인하며, 필요한 확장 기능을 사용할 수 있도록 도와주는 작업 공간이다.

웹 개발 환경을 구성하기 위해 필요한 요소는 다음과 같다.

| 요소 | 설명 |
|---|---|
| IDE | 코드를 작성하고 실행할 수 있는 통합 개발 환경 |
| 프로젝트 폴더 | 소스 코드를 관리하는 작업 폴더 |
| 웹 서버 | HTML, CSS, JavaScript 같은 정적 리소스를 브라우저에 제공 |
| 확장 프로그램 | 개발 편의성을 높여주는 추가 기능 |

이번 실습에서는 VS Code를 이용해 프론트엔드 개발 환경을 구성했다.

## 2. IDE

IDE는 Integrated Development Environment의 약자이다.  
한국어로는 통합 개발 환경이라고 한다.

IDE는 개발자가 코드를 작성하고 실행할 수 있도록 여러 기능을 제공한다.

대표적인 기능은 다음과 같다.

- 코드 작성
- 코드 자동 완성
- 파일 관리
- 실행 및 빌드
- 디버깅
- 터미널 사용
- 확장 기능 설치

대표적인 개발 도구는 다음과 같다.

| 도구 | 설명 |
|---|---|
| Eclipse | Java 개발에서 전통적으로 많이 사용되는 IDE |
| IntelliJ | Java, Spring 개발에서 많이 사용되는 IDE |
| VS Code | 다양한 언어와 확장 기능을 지원하는 코드 편집기 |

Java 기반 개발에서는 Eclipse나 IntelliJ를 많이 사용한다.  
하지만 HTML, CSS, JavaScript 실습이나 React 개발에서는 VS Code도 많이 사용된다.

## 3. VS Code

VS Code는 Visual Studio Code의 줄임말이다.

VS Code는 특정 언어 전용 IDE라기보다는 다양한 언어를 지원하는 코드 편집기에 가깝다.  
필요한 확장 프로그램을 설치하면 HTML, CSS, JavaScript, Java, Python, React 등 다양한 개발 환경을 구성할 수 있다.

VS Code의 장점은 다음과 같다.

- 설치가 비교적 간단하다.
- 가볍게 사용할 수 있다.
- 다양한 확장 프로그램을 제공한다.
- HTML, CSS, JavaScript 작성에 적합하다.
- React, Vue 같은 프론트엔드 개발에도 활용할 수 있다.
- 터미널과 파일 탐색기를 함께 사용할 수 있다.

이번 실습에서는 VS Code를 이용해 정적 웹 페이지를 작성하고 실행하는 환경을 구성했다.

## 4. VS Code 설치 흐름

VS Code 설치 과정은 일반적인 프로그램 설치 과정과 비슷하다.

기본 흐름은 다음과 같다.

```text
VS Code 다운로드
→ 설치 파일 실행
→ 라이선스 동의
→ 설치 경로 확인
→ 추가 옵션 선택
→ 설치 진행
→ VS Code 실행
```

설치 중 선택할 수 있는 옵션에는 다음과 같은 항목들이 있다.

- 바탕화면 바로가기 생성
- 지원되는 파일 형식을 VS Code로 열기
- PATH 등록
- 시작 메뉴 등록

PATH를 등록하면 터미널에서도 VS Code를 실행할 수 있다.

## 5. 프로젝트 폴더 구성

코드를 작성할 때는 소스 파일을 관리할 폴더를 먼저 만드는 것이 좋다.

이번 실습에서는 다음과 같은 구조로 폴더를 구성했다.

```text
vscode_project
└── front
    └── web
```

이 구조는 VS Code에서 웹 실습 파일을 관리하기 위한 기본 작업 공간이다.

이후 `web` 폴더 안에 HTML, CSS, JavaScript 파일을 추가할 수 있다.

예상 폴더 구조는 다음과 같다.

```text
web
├── login.html
├── welcome.html
├── css
│   └── login.css
└── script
    └── login.js
```

각 폴더와 파일의 역할은 다음과 같다.

| 파일 / 폴더 | 역할 |
|---|---|
| `login.html` | 로그인 화면 구조 작성 |
| `welcome.html` | 로그인 성공 후 이동할 화면 |
| `css` | CSS 파일 관리 |
| `script` | JavaScript 파일 관리 |
| `login.css` | 로그인 화면 스타일 작성 |
| `login.js` | 로그인 화면 동작 작성 |

HTML, CSS, JavaScript 파일을 역할별로 분리하면 코드 관리가 쉬워진다.

## 6. VS Code에서 폴더 열기

VS Code에서 프로젝트를 시작하려면 작업할 폴더를 열어야 한다.

흐름은 다음과 같다.

```text
VS Code 실행
→ File
→ Open Folder
→ 작업할 폴더 선택
→ Trust 여부 확인
→ 프로젝트 폴더 열기
```

폴더를 열면 VS Code의 Explorer 영역에서 프로젝트 파일과 폴더를 확인할 수 있다.

이후 해당 폴더 안에서 HTML, CSS, JavaScript 파일을 생성하고 수정할 수 있다.

## 7. 정적 리소스

정적 리소스는 요청할 때마다 내용이 바뀌지 않는 파일을 의미한다.

대표적인 정적 리소스는 다음과 같다.

- HTML
- CSS
- JavaScript
- 이미지 파일

이번 실습에서는 HTML, CSS, JavaScript 파일을 사용했다.  
이 파일들은 서버에서 복잡한 비즈니스 로직을 처리하는 동적 리소스가 아니라, 브라우저에서 해석되어 화면을 구성하는 정적 리소스이다.

정적 리소스는 일반적으로 웹 서버를 통해 클라이언트에게 제공된다.

## 8. 웹 서버와 Live Server

웹 페이지를 브라우저에서 실행하려면 서버가 필요하다.

실제 서비스에서는 Apache, Nginx 같은 웹 서버를 사용할 수 있다.  
하지만 로컬 실습에서는 간단하게 VS Code 확장 프로그램인 Live Server를 사용할 수 있다.

Live Server는 로컬 환경에서 웹 서버를 실행할 수 있도록 도와주는 확장 프로그램이다.

Live Server의 역할은 다음과 같다.

- 로컬 웹 서버 실행
- HTML 파일을 브라우저에서 실행
- 파일 변경 시 빠르게 화면 확인
- 정적 리소스 테스트 지원

Live Server를 사용하면 브라우저에서 다음과 같은 주소로 페이지를 확인할 수 있다.

```text
http://127.0.0.1:5500/login.html
```

각 요소의 의미는 다음과 같다.

| 요소 | 의미 |
|---|---|
| `127.0.0.1` | 내 컴퓨터를 의미하는 localhost |
| `5500` | Live Server가 사용하는 포트 번호 |
| `login.html` | 실행 중인 HTML 파일 |

## 9. localhost

localhost는 현재 사용 중인 내 컴퓨터를 의미한다.

즉, `127.0.0.1`은 외부 서버가 아니라 내 컴퓨터 안에서 실행 중인 서버를 가리킨다.

로컬 환경에서 Live Server를 실행하면 브라우저는 내 컴퓨터에서 실행 중인 서버에 요청을 보내고, 서버는 HTML 파일을 응답한다.

흐름은 다음과 같다.

```text
브라우저
→ localhost 서버에 요청
→ Live Server가 HTML 응답
→ 브라우저가 화면 렌더링
```

실습은 로컬에서 진행하지만, 웹의 기본 구조는 실제 서비스와 같다.  
브라우저가 URL을 통해 서버에 요청하고, 서버가 응답을 반환하는 구조로 동작한다.

## 10. VS Code 확장 프로그램

VS Code는 확장 프로그램을 설치해 기능을 추가할 수 있다.

이번 실습에서 사용한 확장 프로그램은 다음과 같다.

| 확장 프로그램 | 역할 |
|---|---|
| Live Server | 로컬 웹 서버 실행 |
| Material Icon Theme | 파일과 폴더 아이콘을 보기 좋게 표시 |

Live Server는 HTML 파일을 브라우저에서 실행하는 데 사용한다.  
Material Icon Theme은 파일과 폴더를 아이콘으로 구분하기 쉽게 만들어준다.

확장 프로그램은 VS Code 좌측의 Extensions 메뉴에서 검색하고 설치할 수 있다.

## 11. HTML 파일 생성

프로젝트 폴더를 연 뒤 HTML 파일을 생성할 수 있다.

예를 들어 로그인 페이지를 만들기 위해 다음 파일을 생성한다.

```text
login.html
```

VS Code에서는 `html:5` 또는 HTML 기본 템플릿을 사용해 기본 구조를 빠르게 만들 수 있다.

HTML 기본 구조는 다음과 같다.

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Login Page</title>
</head>
<body>

</body>
</html>
```

HTML 문서는 크게 `head`와 `body` 영역으로 나뉜다.

| 영역 | 역할 |
|---|---|
| `head` | 문서 정보, 제목, CSS 연결 등 화면에 직접 보이지 않는 정보 |
| `body` | 브라우저 화면에 실제로 표시되는 내용 |

## 12. CSS와 JavaScript 폴더 분리

웹 페이지를 만들 때 HTML, CSS, JavaScript를 같은 위치에 모두 두면 관리가 어려워질 수 있다.

따라서 역할별로 폴더를 분리하는 것이 좋다.

예시는 다음과 같다.

```text
web
├── login.html
├── css
│   └── login.css
└── script
    └── login.js
```

분리해서 관리하면 다음과 같은 장점이 있다.

- 파일의 역할이 명확해진다.
- 유지보수가 쉬워진다.
- CSS와 JavaScript를 다른 페이지에서도 재사용할 수 있다.
- 프로젝트 구조를 파악하기 쉽다.

## 13. 전체 개발 환경 구성 흐름

이번 실습의 개발 환경 구성 흐름은 다음과 같다.

```text
VS Code 설치
→ 프로젝트 폴더 생성
→ VS Code에서 폴더 열기
→ Live Server 설치
→ Material Icon Theme 설치
→ HTML 파일 생성
→ CSS 폴더 생성
→ script 폴더 생성
→ Live Server로 실행
```

이 과정을 통해 로컬 환경에서 간단한 웹 페이지를 작성하고 브라우저에서 실행할 수 있는 기본 환경을 구성할 수 있다.

## 14. 정리

이번 학습에서는 웹 개발을 위한 기본 개발 환경을 구성했다.

핵심 개념은 다음과 같다.

- 웹 개발에는 코드를 작성하고 실행할 수 있는 IDE가 필요하다.
- VS Code는 다양한 확장 기능을 제공하는 코드 편집기이다.
- 웹 프로젝트는 HTML, CSS, JavaScript 파일을 역할별로 분리해서 관리하는 것이 좋다.
- 정적 리소스는 HTML, CSS, JavaScript, 이미지처럼 요청마다 내용이 바뀌지 않는 파일이다.
- Live Server를 사용하면 로컬 환경에서 웹 서버를 실행할 수 있다.
- localhost는 현재 내 컴퓨터를 의미한다.
- 로컬에서 실행하더라도 브라우저가 서버에 요청하고 응답을 받는 웹의 기본 구조는 동일하다.
