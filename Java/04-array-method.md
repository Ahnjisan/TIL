# Java 배열과 메서드

## 1. 배열이란?

배열은 같은 타입의 여러 값을 하나의 이름으로 관리할 수 있는 자료구조이다.

변수 하나에는 보통 하나의 값만 저장할 수 있다.

예를 들어 학생 점수 3개를 저장하려면 다음과 같이 변수를 여러 개 만들 수 있다.

    int score1 = 90;
    int score2 = 80;
    int score3 = 70;

하지만 저장해야 하는 값이 많아지면 변수를 하나씩 만드는 방식은 관리가 어렵다.

이때 배열을 사용하면 같은 종류의 데이터를 하나의 이름으로 묶어서 관리할 수 있다.

    int[] scores = {90, 80, 70};

## 2. 배열이 필요한 이유

배열을 사용하면 여러 값을 효율적으로 관리할 수 있다.

예를 들어 학생 30명의 점수를 저장해야 한다면 변수 30개를 만드는 것보다 배열 하나를 사용하는 것이 더 적절하다.

배열의 장점은 다음과 같다.

| 장점 | 설명 |
|---|---|
| 데이터 묶음 관리 | 같은 타입의 여러 값을 하나로 관리 |
| 반복문과 함께 사용 가능 | 배열 요소를 반복적으로 처리 가능 |
| 코드 감소 | 여러 변수를 따로 선언하지 않아도 됨 |
| 인덱스 접근 | 순서를 기준으로 값에 접근 가능 |

배열은 반복문과 함께 사용할 때 특히 유용하다.

## 3. 배열 선언

Java에서 배열을 선언하는 기본 형식은 다음과 같다.

    타입[] 배열명;

예시는 다음과 같다.

    int[] scores;
    String[] names;

또는 다음과 같은 방식도 가능하다.

    int scores[];

하지만 Java에서는 보통 `int[] scores` 형태를 더 많이 사용한다.

## 4. 배열 생성

배열은 선언 후 생성해야 사용할 수 있다.

배열 생성에는 `new` 키워드를 사용한다.

기본 형식은 다음과 같다.

    배열명 = new 타입[크기];

예시는 다음과 같다.

    int[] scores;
    scores = new int[3];

위 코드는 정수 3개를 저장할 수 있는 배열을 생성한다.

선언과 생성을 동시에 할 수도 있다.

    int[] scores = new int[3];

## 5. 배열 초기화

배열을 만들면서 값을 바로 넣을 수도 있다.

예시는 다음과 같다.

    int[] scores = {90, 80, 70};

    String[] names = {"Kim", "Lee", "Park"};

이 방식은 배열의 크기와 값을 동시에 정한다.

`scores` 배열은 3개의 값을 가지고, `names` 배열도 3개의 값을 가진다.

## 6. 배열의 인덱스

배열의 각 값은 인덱스를 통해 접근한다.

Java 배열의 인덱스는 0부터 시작한다.

예를 들어 다음 배열이 있다고 가정한다.

    int[] scores = {90, 80, 70};

각 값의 인덱스는 다음과 같다.

| 인덱스 | 값 |
|---|---|
| 0 | 90 |
| 1 | 80 |
| 2 | 70 |

첫 번째 값을 가져오려면 다음과 같이 작성한다.

    scores[0]

두 번째 값을 가져오려면 다음과 같이 작성한다.

    scores[1]

## 7. 배열 값 변경

배열의 값은 인덱스를 사용해서 변경할 수 있다.

예시는 다음과 같다.

    int[] scores = {90, 80, 70};

    scores[1] = 85;

위 코드 실행 후 배열의 값은 다음과 같이 변경된다.

    {90, 85, 70}

즉, 인덱스 1번 위치의 값이 80에서 85로 변경된다.

## 8. 배열 길이

배열의 길이는 `length`를 사용해서 확인할 수 있다.

예시는 다음과 같다.

    int[] scores = {90, 80, 70};

    System.out.println(scores.length);

위 코드는 배열의 길이인 3을 출력한다.

배열의 마지막 인덱스는 `배열.length - 1`이다.

예를 들어 길이가 3인 배열의 마지막 인덱스는 2이다.

## 9. 배열과 for문

배열은 반복문과 함께 사용할 때 효율적이다.

예시는 다음과 같다.

    int[] scores = {90, 80, 70};

    for (int i = 0; i < scores.length; i++) {
        System.out.println(scores[i]);
    }

위 코드는 배열의 모든 값을 순서대로 출력한다.

반복문의 조건식에 `scores.length`를 사용하면 배열의 길이가 바뀌어도 코드 수정이 줄어든다.

## 10. 향상된 for문

Java에서는 배열의 값을 하나씩 꺼내기 위해 향상된 for문을 사용할 수 있다.

기본 형식은 다음과 같다.

    for (타입 변수명 : 배열명) {
        실행할 코드
    }

예시는 다음과 같다.

    int[] scores = {90, 80, 70};

    for (int score : scores) {
        System.out.println(score);
    }

향상된 for문은 배열의 모든 값을 순서대로 읽을 때 편리하다.

다만 인덱스가 필요한 경우에는 일반 for문을 사용하는 것이 좋다.

## 11. 배열 사용 시 주의할 점

배열을 사용할 때 가장 주의해야 할 점은 인덱스 범위이다.

배열의 인덱스는 0부터 시작하고, 마지막 인덱스는 `length - 1`이다.

예를 들어 다음 배열의 길이는 3이다.

    int[] scores = {90, 80, 70};

사용 가능한 인덱스는 0, 1, 2이다.

만약 다음과 같이 작성하면 오류가 발생한다.

    scores[3]

인덱스 3은 존재하지 않기 때문이다.

## 12. 메서드란?

메서드는 데이터를 다루는 행위이다.

변수가 데이터를 담는 공간이라면, 메서드는 그 데이터를 사용해서 특정 작업을 수행하는 코드 블록이다.

예를 들어 점수를 전달받아 합격 여부를 판단하거나, 이름을 전달받아 인사말을 출력하는 기능을 메서드로 만들 수 있다.

메서드는 클래스의 구성 요소 중 하나이다.

Java에서 클래스는 변수와 메서드로 구성된다.

## 13. 메서드 선언 형식

메서드의 기본 형식은 다음과 같다.

    접근지정자 반환타입 메서드명(매개변수) {
        실행할 코드
    }

각 요소의 의미는 다음과 같다.

| 요소 | 설명 |
|---|---|
| 접근지정자 | 메서드 접근 범위 |
| 반환타입 | 메서드 실행 후 돌려줄 값의 타입 |
| 메서드명 | 메서드 이름 |
| 매개변수 | 메서드가 전달받는 값 |
| 블록 | 실제 실행할 코드 영역 |

예시는 다음과 같다.

    public int add(int a, int b) {
        return a + b;
    }

위 메서드는 정수 두 개를 받아 더한 뒤 결과를 반환한다.

## 14. 매개변수

매개변수는 메서드가 실행될 때 외부에서 전달받는 값이다.

예시는 다음과 같다.

    public void printName(String name) {
        System.out.println(name);
    }

위 메서드에서 `String name`이 매개변수이다.

메서드를 호출할 때 전달한 값이 매개변수에 들어가고, 메서드 안에서 사용할 수 있다.

## 15. 반환 타입

반환 타입은 메서드가 실행된 뒤 호출한 쪽으로 돌려주는 값의 타입이다.

예를 들어 정수 값을 반환하는 메서드는 반환 타입으로 `int`를 사용한다.

    public int getAge() {
        return 25;
    }

문자열을 반환하는 메서드는 `String`을 사용할 수 있다.

    public String getName() {
        return "Jisan";
    }

반환값이 없다면 `void`를 사용한다.

    public void printHello() {
        System.out.println("Hello");
    }

## 16. return

`return`은 메서드의 실행 결과를 호출한 쪽으로 돌려줄 때 사용한다.

예시는 다음과 같다.

    public int add(int a, int b) {
        return a + b;
    }

위 메서드는 `a + b`의 결과를 반환한다.

반환 타입이 `int`이면 return하는 값도 int 타입이어야 한다.

반환 타입과 실제 반환 값의 타입이 맞지 않으면 오류가 발생할 수 있다.

## 17. void

`void`는 반환값이 없다는 의미이다.

예시는 다음과 같다.

    public void printMessage(String message) {
        System.out.println(message);
    }

이 메서드는 문자열을 전달받아 출력하지만, 호출한 쪽으로 값을 반환하지는 않는다.

즉, 실행 결과를 돌려줄 필요가 없는 메서드에서는 `void`를 사용한다.

## 18. 메서드의 네 가지 유형

메서드는 매개변수와 반환값의 유무에 따라 네 가지 유형으로 나눌 수 있다.

| 유형 | 매개변수 | 반환값 | 예시 |
|---|---|---|---|
| 1 | 있음 | 있음 | 두 수를 받아 합계 반환 |
| 2 | 있음 | 없음 | 이름을 받아 출력 |
| 3 | 없음 | 있음 | 현재 시간 반환 |
| 4 | 없음 | 없음 | 안내 문구 출력 |

예시는 다음과 같다.

    public int add(int a, int b) {
        return a + b;
    }

    public void printName(String name) {
        System.out.println(name);
    }

    public String getMessage() {
        return "Hello";
    }

    public void printHello() {
        System.out.println("Hello");
    }

## 19. main 메서드

`main` 메서드는 Java 프로그램의 실행 시작점이다.

기본 구조는 다음과 같다.

    public static void main(String[] args)

Java 프로그램을 실행하면 JVM은 main 메서드부터 실행한다.

main 메서드가 없는 클래스는 컴파일은 가능할 수 있지만 직접 실행할 수는 없다.

즉, main 메서드는 Java 프로그램의 엔트리 포인트이다.

## 20. 배열을 매개변수로 전달하기

배열도 메서드의 매개변수로 전달할 수 있다.

예시는 다음과 같다.

    public int sum(int[] numbers) {
        int total = 0;

        for (int number : numbers) {
            total += number;
        }

        return total;
    }

위 메서드는 정수 배열을 전달받아 모든 값을 더한 뒤 합계를 반환한다.

배열을 매개변수로 사용하면 여러 값을 한 번에 메서드로 전달할 수 있다.

## 21. 배열을 반환하는 메서드

메서드는 배열을 반환할 수도 있다.

예시는 다음과 같다.

    public int[] createScores() {
        int[] scores = {90, 80, 70};
        return scores;
    }

반환 타입이 `int[]`이면 정수 배열을 반환한다는 의미이다.

배열도 참조 타입이기 때문에 메서드의 매개변수나 반환 타입으로 사용할 수 있다.

## 22. 배열과 메서드를 함께 사용하는 이유

배열과 메서드를 함께 사용하면 여러 데이터를 효율적으로 처리할 수 있다.

예를 들어 점수 배열을 전달받아 합계나 평균을 계산하는 메서드를 만들 수 있다.

흐름은 다음과 같다.

    점수 배열 생성
    → 배열을 메서드에 전달
    → 메서드에서 반복문으로 배열 처리
    → 결과 반환

예시는 다음과 같다.

    public double average(int[] scores) {
        int total = 0;

        for (int score : scores) {
            total += score;
        }

        return total / (double) scores.length;
    }

이처럼 배열은 여러 값을 관리하고, 메서드는 그 값을 처리하는 역할을 한다.

## 23. 정리

이번 학습에서는 Java의 배열과 메서드를 정리했다.

핵심 개념은 다음과 같다.

- 배열은 같은 타입의 여러 값을 하나의 이름으로 관리하는 자료구조이다.
- 배열은 `타입[] 배열명` 형식으로 선언한다.
- 배열은 `new 타입[크기]` 형식으로 생성한다.
- 배열은 `{값1, 값2, 값3}` 형식으로 초기화할 수 있다.
- 배열의 인덱스는 0부터 시작한다.
- 배열의 길이는 `배열명.length`로 확인한다.
- 배열의 마지막 인덱스는 `length - 1`이다.
- 배열은 for문과 함께 사용할 때 효율적이다.
- 향상된 for문은 배열의 값을 순서대로 읽을 때 편리하다.
- 메서드는 데이터를 다루는 행위이다.
- 메서드는 접근지정자, 반환타입, 메서드명, 매개변수, 블록으로 구성된다.
- 매개변수는 메서드가 전달받는 값이다.
- 반환 타입은 메서드가 돌려주는 값의 타입이다.
- `return`은 값을 반환할 때 사용한다.
- `void`는 반환값이 없다는 의미이다.
- 메서드는 매개변수와 반환값의 유무에 따라 네 가지 유형으로 나눌 수 있다.
- 배열도 메서드의 매개변수나 반환 타입으로 사용할 수 있다.
